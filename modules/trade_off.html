Trade-off between Time & Space
  • Algorithm uses Mmemory & Time to complete a task.
      • In most cases, we can have trade-off between time and space. i.e we can solve a problem,
      • Either in less time by using more memory
      • Or in less Memory by using more Time
  • The choice depends on the constrains of a problem
    • Ex 1.: Merge sort algorithm is exceedingly fast but requires lot of space.
      But, Bubble sort is exceedingly slow but requires less space.
    • Ex 2.: We can store already calculated results in some recursion problems
      Instead of calculating them multiple times (e.x: Fibonocci problem)
